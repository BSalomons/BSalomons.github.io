Deal Pipeline Summary

There are three objects associated with the deal pipeline within Salesforce: 1) Tenant/Buyer Leads, 2) In Contract, and 3) Closed Deals. There are buttons within Salesforce that advance a transaction to the next object. What these buttons more or less do is copies the data within one object, creates a new record in the next object, pastes that data into the newly created record, and updates the record in the original object accordingly.

For example, hitting “Close Deal” within the In Contract object will create a new Closed Deal record, put all the transaction data into this new record, and then update the In Contract to be “sold” or “leased”. Our various filters, forms and reports work off the values of these fields.

The In Contract object is the most important as the data in this object is used for accounting and auditing purposes with the brokerage. It is imperative that the data entered into this object is both accurate and up to date.

In Contract has various sub levels within it that are regulated with the “Status” field. The first is “Negotiating” which means offers have been presented but have not been accepted yet. This is the first step of our process which assigns a deal number to a file. The second is “Pending” which means an offer has been accepted by two parties and are now working through conditions precedent. The third is “In Contract” which is the interim period when all conditions precedent have been waived or satisfied and the transaction is proceeding to close. The fourth is “Closed” and is accomplished by simply hitting the “Close Deal” button within the record. Only the broker handles this process. The fifth option is “Dead” which means the transaction has collapsed at some point during one of the above states.

Tenant/Buyer Leads is an object for agents to keep track of any leads that are searching for properties and provides them a space to enter important information. It also allows you to put them in a “hold” status which means nothing works for them in the market right now but you want to keep them in mind should something come available in the future.

Closed Deals is an object for tracking comps so that you can filter and see what properties have sold and leased for. There is the ability to track external transactions that are brokerage was not involved in.
