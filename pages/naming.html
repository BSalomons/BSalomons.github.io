Naming Guidelines for Files, Folders, and Programs

The following information details how files, Listings, Properties, folders, etc. across software platforms are to be followed.

K Drive

Executed Documents

  • Any documents (offers, leases, notices, listing agreements, etc) that are fully executed by all parties must have the word “Executed” at the start
  • The date of execution should also be included in the file name
  • The location of these documents should either be in their own folder marked as such or at the root level with no other files cluttering them

Folders

  • Should be organized so that similar documents are stored together
  • For example, titles and encumbrances; floor plans; etc.
  • Offers and other documents should be stored together in a common folder

Listing Folders

  • Must be named “Seller/Landlord – Address of the Property”
  • Seller/Landlord can be a person or company

Offer Folders

  • Must be name “Deal Number-Seller/Landlord – Buyer/Tenant”
  • Deal number is autogenerated in Salesforce and must be copied to here
  • The names used for Seller/Landlord and Buyer/Tenant should be descriptive and not numbered companies. These will be much easier to locate and reference in the future if done this way. Using the deal number can quickly look up the file through the search bar

Database

Properties

  • Must use the Property’s address as the name.
  • This is the easiest way to search out properties in the database and Salesforce

Listings

  • Must be named “Seller/Landlord – Address of the Property”
  • This is the easiest way to search out properties in the database and Salesforce

Salesforce

In Contract

  • Must have “Seller/Landlord – Buyer/Tenant” and use the legal entities in the contract
  • This differs from the file folders because this name is used for auditing purposes and must be what is contained in the contract.
  • If there is more than one party on one side, use “et al” after the first name which is the legal meaning “and others” in Latin
