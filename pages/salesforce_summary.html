Salesforce Summary

Salesforce is a platform used to track various aspects for the Brokerage. The primary functions of the Salesforce platform are:

  • The Database synchronizes its Properties and Listings records with the Salesforce database. In this instance, Salesforce acts as an external database.
  • It is used as a contact database which synchronizes with our Google apps (Gmail, Contacts, Calendar) so that everyone is using the same contact list. This uses the Contacts object (child) and the Companies object (parent).
  • It tracks and keeps record of our Deal Pipeline. It starts with the Tenant/Buyer Leads object, then transitions into the In Contract object, then finishes with the Closed Deals
  • Property Management details are entered and tracked

All of these objects interact with each other and can be searched and referenced accordingly. For example, one single Contact can be all of the following:

  • a client for a Listing
  • an owner of a Property
  • a potential buyer
  • a tenant in an In Contract deal
  • the seller in a Closed Deal
  • the owner of a Property Management

When going to this Contact record, it will show all of the other objects that they are associated with. This is what makes it a relational database.

For the Properties and Listings object only, it is a one-way sync between the Database and Salesforce, meaning that if you edit a record in Salesforce, it will not sync with the Database and that data will be lost if the same record in the Database is saved. This is because the data entered in the Database overwrites anything saved in Salesforce.

For the above reasons, it is very important to accurately add and update the data contained within Salesforce frequently. Everyone works off the same data and this information is used to send out mail, deposits, cheques, etc.

Within each individual record of Salesforce, there is the details associated with the record and then “Related Lists”.  Related Lists are the relationships shown with other objects and this particular record. Typically, they are displayed at the bottom of the page except for Properties in which they are their own tab near the top. For example, navigating to a Contact would show which Properties they own, which Listings we have with them, and which Closed Deals they have been involved in.
