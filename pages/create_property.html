Create a New Property

A Property represents anything that has a certificate of title. Most of the time it is the piece of land but the one exception are condominiums. If it has a title, then there needs to be its own Property record. A Listing can only have one Property record associated with it.

The Property is the highest parent record which can have further child records below it. What this means is that a child record can’t exist unless the parent does. You can’t have a building with out a piece of land to put it on. You also can’t have a Premises within a building. Therefore, the Property is a parent of a Building record which is the parent to the Premises record (Property is the grandparent). You need to create the parent records first (and save it to the Database) before you can add and associate the child records with it.

If you have checked and are sure the Property does not exist in the Database already, then follow these steps:

  1. Navigate to the Property page
  2. Click either one of the Record Types at the top (Add Office, Add Industrial, ….) or click the green “Add Property” button. The difference between the two is the top categories will choose the record type chosen whereas the green button will just load the blank form and a record type will then need to be selected.

Complete the fields within the form. The fields needing explaining are below, obvious ones are not included:

  • Enter the Address for the Property
  • 4909 50 Street
  • Must follow this nomenclature
  • Choose the applicable Record Type
  • If multiple uses, choose the best or most applicable
  • This field is used with the filters on the front end of the website so if this is not chosen correctly then people won’t see it. Think about how you would classify it if you were a user on the front end

At this point, click the “Save and Continue Editing” button on the right-hand side before proceeding as the Record Type changes what the form looks like depending on selection.

  • Use this picklist to choose other uses for the site
  • For example if Land was chosen for the Record Type, then choose what kind of Land
  • If the Property can be used for Office but is primarily Retail, then add Office
  • Do not use this unless necessary. We don’t want every property falling under every category. Just because office can go into a retail mall doesn’t mean it should be included here. If you are on the front end and you select Office, that means you are looking for Office, not a Retail space.
  • This should be zoning so select the correct zoning for the property by typing what it is.
  • There is the off chance it doesn’t exist, if so click here to add it to the Database
  • Choose the appropriate Municipality

At this point, click the “Save and Continue Editing” button on the right-hand side before proceeding.

  • You must add the leading zeros and it must be 10 digits long
  • This is a unique number and no two properties can have the same LINC number at the Land Titles Office
  • The Existing Parcel checkbox is enabled by default. Do not uncheck this unless the property is proposed and hasn’t been created yet. Ie needs to be subdivided
  • Old fields that aren’t really used anymore. Feel free to skip

  • Anything unique about the property can be added here. Anything that is put here will show up in a keyword search
  • This is how the map on the front end places a pin for the property and must be completed
  • On the left hand side of the map there is a “pin” button, click it
  • Zoom in, pan and move the map to place the pin in the correct spot then click the mouse

At this point, click the “Save and Continue Editing” button on the right-hand side before proceeding.

  • Drag and drop multiple images in the space to upload
  • This is what displays on the front end of the website
  • Generally you are trying to keep just pictures of the property here and leave pictures of a specific unit or Premises under that record
  • This is what is downloadable off the website
  • Do not put anything confidential here as the public will see it
  • This is for our own private use and anything can be uploaded
  • Rival brochures should be uploaded here to refer back to at later dates
  • Same with titles
  • Entering either one of these fields will automatically update the other one upon saving
  • Old field, ignore
  • This is the amount of parking for the entire site
  • This is at the very bottom of the page
  • This was functioning properly but since we’ve done some changes to the code, it no longer works to edit or add new shapes
  • Leave whatever the default is here

At this point, click the “Save and Continue Editing” button on the right-hand side before proceeding.

If just Land, then the Property is complete. If there are buildings on the site, then complete the Building form below. Even though it falls under the Property page, this technically is its own class of records; we have just embedded the form within the Property page to make it easier for data input (same with Premises).

Click here on how to Add a Building

The Building information must be completed. The fields under “Site Information” are calculated based on the values entered for the Buildings and the front end filters use these values. Therefore, if there are two buildings then the total Site Rentable Area is the sum of the two building’s rentable areas (ditto with the other fields).

The Building information must be completed first prior to completing any Premises information. This is because the Building record is parent to the Premises child record (a Premises can’t exist without the Building).
